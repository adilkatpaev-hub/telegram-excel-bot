# PRD: Telegram-бот для заполнения Excel-отчета

## 1. Назначение проекта

Проект представляет собой Telegram-бота, который принимает от пользователя информацию о выполненной работе и автоматически записывает ее в Excel-таблицу отчета.

Основная цель проекта - упростить сбор ежедневных рабочих отчетов и убрать ручное заполнение Excel-файла.

## 2. Проблема

Сейчас отчет может заполняться вручную: сотрудник отправляет информацию сообщением, после чего данные нужно переносить в Excel. Это занимает время и может приводить к ошибкам:

- пропущенные строки;
- неправильный порядок данных;
- разные форматы заполнения;
- потеря сообщений;
- лишняя ручная работа.

Telegram-бот должен стандартизировать ввод данных и сразу сохранять их в таблицу.

## 3. Целевая аудитория

Основные пользователи:

- сотрудники, которые выполняют работы на объектах;
- мастер, инженер или руководитель, который собирает отчеты;
- администратор, который ведет итоговый Excel-файл.

## 4. Основной сценарий использования

1. Пользователь открывает Telegram-бота.
2. Пользователь нажимает или отправляет команду `/start`.
3. Бот по очереди задает вопросы:
   - фамилия;
   - дата;
   - наименование устройства или адреса;
   - проделанная работа.
4. Пользователь отвечает на каждый вопрос.
5. После последнего ответа бот добавляет новую строку в Excel-таблицу.
6. Бот сообщает пользователю, что отчет сохранен.

## 5. Поля Excel-отчета

Excel-файл должен содержать следующие колонки:

| Колонка | Описание | Пример |
| --- | --- | --- |
| Фамилия | Фамилия сотрудника, который отправляет отчет | Иванов |
| Дата | Дата выполнения работы | 07.07.2026 |
| Наименование устройства или адреса | Название оборудования, объекта или адрес | Шкаф управления 2 / ул. Абая 15 |
| Проделанная работа | Краткое описание выполненной работы | Проверил питание, заменил кабель |

## 6. Функциональные требования

### 6.1 Telegram-бот

Бот должен:

- принимать команду `/start`;
- начинать заполнение нового отчета;
- задавать вопросы пользователю по очереди;
- сохранять ответы пользователя;
- после завершения записи добавлять данные в Excel;
- отправлять подтверждение об успешном сохранении;
- поддерживать команду отмены заполнения.

### 6.2 Заполнение отчета

Бот должен запрашивать данные строго в следующем порядке:

1. Фамилия.
2. Дата.
3. Наименование устройства или адреса.
4. Проделанная работа.

После получения всех данных бот должен создать новую строку в Excel-файле.

### 6.3 Excel-файл

Система должна:

- создавать Excel-файл автоматически, если его еще нет;
- создавать лист с отчетом;
- добавлять заголовки колонок;
- добавлять каждую новую заявку в новую строку;
- не удалять старые записи;
- сохранять файл после каждого нового отчета.

### 6.4 Команда отмены

Пользователь должен иметь возможность отменить заполнение отчета командой `/cancel`.

После отмены бот должен очистить временно введенные данные и предложить начать заново через `/start`.

## 7. Нефункциональные требования

### 7.1 Простота использования

Пользователь не должен заполнять сложные формы. Весь процесс должен проходить в Telegram через обычные сообщения.

### 7.2 Надежность

Данные не должны теряться после завершения заполнения. Каждая завершенная запись должна попадать в Excel-файл.

### 7.3 Понятность

Сообщения бота должны быть короткими и понятными:

- "Введите фамилию:"
- "Введите дату:"
- "Введите наименование устройства или адрес:"
- "Введите проделанную работу:"
- "Готово, отчет сохранен."

### 7.4 Локальное хранение

На первом этапе Excel-файл хранится локально на компьютере или сервере, где запущен бот.

## 8. Границы первой версии

В первую версию не входит:

## 14. Требования к запуску

Для работы проекта понадобится:

- Telegram-бот, созданный через BotFather;
- токен Telegram-бота;
- компьютер или сервер, где бот будет постоянно запущен;
- установленный Python;
- файл Excel для хранения отчетов.

## 15. Критерии готовности

Проект считается готовым, если:

- бот запускается без ошибок;
- пользователь может начать отчет через `/start`;
- бот задает все 4 вопроса;
- ответы пользователя сохраняются;
- после завершения появляется новая строка в Excel;
- старые строки в Excel не удаляются;
- пользователь получает подтверждение о сохранении;
- заполнение можно отменить через `/cancel`.

## 16. Риски

| Риск | Описание | Решение |
| --- | --- | --- |
| Бот выключен | Если компьютер или сервер выключен, бот не принимает отчеты | Запускать бота на сервере или постоянно включенном ПК |
| Excel-файл открыт | Если файл открыт в Excel, запись может быть заблокирована | Не держать файл открытым во время работы бота или использовать базу данных |
| Ошибка в дате | Пользователь может ввести дату в разном формате | Добавить проверку формата даты |
| Посторонние пользователи | Любой, кто найдет бота, может отправить отчет | Добавить список разрешенных Telegram ID |

## 17. Итог

Проект должен дать простой инструмент для сбора отчетов через Telegram. Пользователь отправляет данные в привычном мессенджере, а руководитель получает аккуратно заполненный Excel-файл с одинаковыми колонками и единым порядком данных.

## 19. Порядок запуска

Для запуска проекта нужно:

1. Открыть папку проекта:

```text
C:\Users\pc\Documents\ADIL project
```

2. Один раз запустить:

```text
install.bat
```

3. После установки запускать бота через:

```text
start_bot.bat
```

4. Не закрывать открывшееся окно запуска. Пока окно открыто, бот работает.

5. В Telegram открыть:

```text
@adil_report_excel_bot
```

6. Отправить команду:

```text
/start
```

После этого бот должен начать задавать вопросы для заполнения отчета.

## 22. Возможные проблемы и решения

| Проблема | Причина | Решение |
| --- | --- | --- |
| Бот не отвечает на `/start` | Программа бота не запущена | Запустить `start_bot.bat` |
| Бот перестал отвечать | Окно запуска закрыто | Снова открыть `start_bot.bat` |
| Ошибка `BOT_TOKEN` | Нет файла `.env` или токен записан неправильно | Проверить строку `BOT_TOKEN=...` в `.env` |
| Ошибка подключения к Telegram | Нет интернета, Telegram заблокирован сетью или firewall | Проверить интернет, VPN/firewall, разрешить Python доступ в сеть |
| Команда `python` не работает | Windows запускает заглушку Microsoft Store | Использовать установленный Python или запуск через `.venv` |
| Excel-файл не обновляется | `report.xlsx` открыт в Excel и заблокирован | Закрыть файл Excel и повторить отправку отчета |
